If you run your script twice, does your database end up with duplicate rows? How did you prevent 
that?

Answer:
No, the dataset does not contain duplicate rows even if the script is executed multiple times. 
This is because duplicate handling is properly implemented in the save_data() function inside 
data_pipeline.py.

How duplicates are prevented:
Every time the script runs, it first loads the existing dataset from the CSV file:
existing_data = load_existing_data()

Then it appends the newly fetched data with the existing data:
combined_data = pd.concat([existing_data, new_data], ignore_index=True)

After merging both datasets, duplicates are removed using specific columns:
combined_data.drop_duplicates(subset=["title", "source"], inplace=True)

Finally, the cleaned dataset is saved back to the CSV file:
combined_data.to_csv("news_data.csv", index=False)

Why this approach works:
Duplicates are checked using the title + source combination, so even if the same 
article is fetched again from the API, it won’t be stored twice. 
This keeps the dataset clean, consistent, and free from duplicates, and allows the 
script to run multiple times safely without data corruption.
